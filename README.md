## **关键执行要点（西蒙×笨办法）**

### 1. **严格遵循“笨办法”的要求**
```
- 必须手敲每一个字符
- 运行并确保没有错误
- 仔细研究每一个错误信息
- 完成所有“附加练习”
- 在代码中写注释解释每一行
```

### 2. **西蒙式的每日专注流程**
```
上午（90分钟）：
- 学习新的练习
- 逐行手敲代码
- 运行并调试

下午（90分钟）：
- 完成所有附加练习
- 尝试修改代码（改变变量、逻辑）
- 不看书写出相似功能的代码

晚上（30分钟）：
- 整理今日代码到GitHub
- 写学习日志（今天学到什么，遇到什么问题）
- 预习明天的练习
```

### 3. **刻意练习策略**
- **第1遍**：照着书敲，理解每一行
- **第2遍**：不看书写，凭记忆敲
- **第3遍**：修改程序功能，添加新特性
- **第4遍**：给别人讲解这个程序

### 4. **错误处理心态**
- **错误是你的朋友**：每个错误信息都是学习机会
- **“硬方式”**：遇到问题先自己思考15分钟
- **三问法则**：问自己“这个错误是什么？为什么会出现？如何修复？”
